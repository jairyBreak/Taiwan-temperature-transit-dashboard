# 台灣交通與氣象動態預測 (Taiwan Transit Dashboard) 🇹🇼🚆

這是一個結合「台北捷運」、「台鐵」與「真實氣象」的互動式地圖視覺化專案。

## ✨ 專案特色

- **16 大重點車站**：收錄西門、台北車站、士林等核心樞紐的真實流量。
- **純靜態架構**：資料已預先處理為 JSON，載入極快，不需依賴後端伺服器。
- **動態視覺化**：點擊地圖站點，自動渲染專屬的氣候與人流對比圖表。

## 🏗️ 專案架構與設計模式

本專案採用**純前端、無伺服器（Serverless）的輕量化架構**，結合地理資訊系統（GIS）與統計學模型，實現氣象與人流的動態模擬與預估。

### 1. 前端狀態管理與架構 (Frontend Core)
* **核心技術**：基於 HTML5、CSS3 (Vanilla CSS) 與 ES6 JavaScript 原生模組，無框架開銷，載入與執行效率極高。
* **狀態驅動 (State Management)**：採用全域狀態物件（`STATE`）管理應用程式生命週期：
  * **模式切換**：`simulation`（模擬調整）、`live`（即時 CWA API）與 `history`（歷史月份平均/模型預測）。
  * **滑桿調整**：儲存氣溫調整值偏差（`tempAdj`），用於連動插值地圖與預測模型。
  * **圖表/地圖實例**：快取 Leaflet 地圖、IDW 圖層與 Chart.js 圖表實例，避免記憶體洩漏與重複渲染。

### 2. GIS 地圖與空間內插技術 (GIS & IDW Engine)
* **底圖整合**：使用 Leaflet.js 整合 CartoDB Dark Matter 暗色系圖磚，符合現代深色美學視覺。
* **反距離加權插值 (IDW, Inverse Distance Weighting)**：
  * 當前全台有 304 個氣象觀測點（包含 280+ 個高海拔山區氣象站與台鐵支線小站作為隱藏觀測點）。
  * 地圖利用 `leaflet-idw.js` 將這 304 個點的實測/模擬溫度進行空間插補，繪製出平滑的全台區域氣溫著色漸層圖層。
* **台灣本島海岸遮罩 (Coastline Masking)**：
  * IDW 內插預設會將溫度渲染至海洋區域。本專案引入了複雜的海岸線多邊形數據（`data/taiwan_mask.js`）。
  * 透過遮罩層的疊加與反向遮蔽，將 IDW 的漸層範圍**精確限制在台灣本島陸地範圍內**，海域則保持底圖的黑灰色，大幅提升地圖的視覺質感與專業度。

### 3. 氣溫人流統計學預測模型 (Statistical Simulation Engine)
* **一元線性迴歸 (Linear Regression)**：
  * 系統載入時，針對全台 26 個主車站（10 個捷運站、16 個台鐵大站）的 120 天日資料進行氣溫（$x$）與人流量（$y$）的一元線性迴歸計算（$y = ax + b$），求出斜率、截距、皮爾森相關係數（$r$）與決定係數（$R^2$），藉此建立各站的氣溫敏感度模型。
* **等比例波動模擬演算法 (Fluctuation Scaling Algorithm)**：
  * 為了避免模擬模式下折線圖只呈現單調的平滑直線，本專案採用了等比例縮放模型：
    $$flow_{simulated} = flow_{actual} \times \left( \frac{simulate(T_{adjusted}, W)}{simulate(T_{actual}, \text{'sunny'})} \right)$$
  * 這套設計保留了 120 天中每日的**實際波動細節**（例如週六/週日人潮高峰、跨年/連假異常），同時又能讓整條人流曲線隨著使用者的溫度調整與天氣按鈕（雨天 -20%、颱風 -90%、冷氣房/低溫避寒調整）進行動態縮放與平移。

### 4. 數據預處理管道 (Data Pipeline)
* 台北捷運每日分時各站 OD 流量與台鐵日進出站人數（每日高達數千萬筆流向記錄）已透過 `backend/` 目錄下的 Python 指令碼進行去重、統計與映射，彙整為各月份輕量化的靜態 JSON 檔案（如 `data/mrt_2026_01.json` 至 `04.json`），使前端只需加載輕量資料即可快速載入 120 天的時間軸數據。

## 🚀 如何觀看與執行

這是一個純前端 (HTML/CSS/JS) 的專案。您只需要：
1. 下載此專案資料夾。
2. 使用任何靜態伺服器（例如 VS Code 的 Live Server 擴充套件，或執行 `python -m http.server 8000`）。
3. 用瀏覽器打開 `http://localhost:8000` 即可觀看。

*(本專案已可直接託管於 GitHub Pages 完美運行)*

## 📄 資料來源

- **臺北捷運每日客流量與分時 OD 流量資料**：來源自 [臺北程式碼與資料開放大平台 (台北資料大平台)](https://data.taipei/)，包含 2026 年 1~4 月捷運各站點每日進出站與 OD 分時統計。
- **臺鐵每日各站進出站乘客人數**：來源自 [政府資料開放平台](https://data.gov.tw/)（由國營臺灣鐵路股份有限公司提供），用於分析全台主要火車站每日運輸量與基準人流。
- **即時氣象實測資料**：來源自 [交通部中央氣象署氣象資料開放平台](https://opendata.cwa.gov.tw/)，串接「自動氣象站-氣象觀測資料 (O-A0003-001)」API 以動態獲取全台各測站當前實測溫度與天氣概況。
- **歷史日均溫氣象資料**：對接 [Open-Meteo 歷史氣候 API](https://open-meteo.com/)，根據全台 300+ 觀測站的精確地理座標（經緯度）獲取 2026 年 1~4 月每日均溫數據，用於繪製空間內插（IDW）的歷史區域等溫線圖層。
